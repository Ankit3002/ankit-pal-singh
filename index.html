PAPPL-based Printer Applications: Option Setting presets via web Interface

Generally, Printer Applications as emulation of driverless IPP printers only support standard IPP job attributes as user-settable options: media size/type/source, duplex, printer-resolution, print-quality, print-content-optimize, … but some drivers, like for example Gutenprint or also PostScript printers, have many options to fine-tune the printout and those cannot get individually mapped to IPP options so that the user can control them in a print dialog. Also, many print dialogs, especially of phones, are limited to standard IPP attributes.

Created a web interface from where one can create and edit any number of presets.

Defined the structure of presets and then wrote APIs to create and edit presets. Additionally, implemented APIs to store and fetch presets from the presets object during runtime.

Subsequently, defined APIs to fetch presets from the state file and store presets in the state file.

After creating all of that, implemented code to add presets to the thread running the printer, and worked on synchronizing the code to ensure smooth execution. Furthermore, added the code to include "job-presets-supported" in the client->response object, which can be called by CPDB enabled backends.
